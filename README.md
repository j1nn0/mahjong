# Mahjong

React Ink と TypeScript で作る、ターミナル上の一人用リーチ麻雀ゲームです。
3人のコンピュータプレイヤーと東風戦（東1局から東4局）を完走できます。
全役の完全対応より、局進行、点数の支払い、供託、本場、親継続、続きからの再開、最終順位まで破綻せずゲームが進むことを優先して v1.0.0 に達しました。

## 実装済みの機能

三人の対戦相手はそれぞれ異なる**AI性格**（攻撃性、リスク許容度、副露指向、リーチ指向、打点志向の5軸）を持ちます。
ゲーム開始前にテンプレートから選ぶか、各パラメータを直接調整して打ち方を設定できます。

基本進行:

- 東風戦（東1局から東4局）
- 南入とサドンデス
- オーラス親トップ終了
- 親継続、本場、供託の持ち越し
- トビ終了（0点未満で即時終了、0点ちょうどは続行）
- 同点タイブレーク（起家に近い席を上位とする）

あがりと役:

- ロン、ツモ、流局、聴牌、聴牌料
- リーチとリーチ後ツモ切り
- チー、ポン、カン（暗槓、加槓、大明槓）
- 食い替え禁止
- ドラ、赤ドラ、裏ドラ、カンドラ
- 一発、海底、河底、嶺上開花、搶槓
- フリテン（自分の捨て牌、同巡見逃し、リーチ後見逃し）
- 七対子、国士無双を含むあがり形判定
- 天和、地和
- 流し満貫
- ダブロン（頭ハネあり、リーチ棒は上家が取得）
- 三家和（途中流局）

特殊ルール:

- 途中流局（九種九牌、四風連打、四家立直、四槓散了）
- 責任払い（大三元と大四喜の包）

AI:

- **コンピュータプレイヤーの自動打牌**: 攻撃性、リスク許容度、副露指向、リーチ指向、打点志向の5軸で打ち分け
- **AI押し引き判断**: 推定打点、和了確率、牌の危険度、性格パラメータから総合判定
- **AI性格設定UI**: ゲーム開始前にテンプレート選択、カスタム調整、ランダム割り当てから選択可能

UI:

- あなたと他家の捨て牌、鳴き、カンした牌の表示
- リーチ宣言牌の強調表示
- 巡目表示（残り巡数、リーチ棒を常時太字表示）
- ターン進行ログ（直近5件の捨て牌履歴）
- シャンテン数と待ち牌の常時表示
- ドラ表示牌と赤ドラの色強調表示
- 選択牌のアンダーラインと太字ハイライト
- 相手リーチ表示の太字黄色化
- 色付きキー凡例（KeyLegend）

永続化:

- 保存済み対戦の続きから再開（`.mahjong-save.json`）

## 非対応

- 半荘（東南戦）
- 西入
- ウマとオカ（順位精算）
- チョンボ

方針の背景は [ADR 0001](docs/adr/0001-finish-east-only-game-before-full-rules.md) と [ADR 0002](docs/adr/0002-expand-east-game-with-common-riichi-rules.md) を参照してください。

## Requirements

- Node.js
- pnpm

## Setup

```bash
pnpm install
```

## Run

```bash
pnpm start
```

初回起動時、または保存データがない場合はAI性格設定画面に進みます。
保存済み対戦がある場合は、続きから再開するか新規対戦を始めるかを選択できます。

AI性格設定画面では、3人のコンピュータプレイヤーの打ち方をテンプレート（バランサー、守りの名人、攻めの一点張り、手役派、鳴きの達人）から選ぶか、各パラメータを 1–5 で個別調整するか、ランダムに割り当てられます。

AI思考遅延は `MAHJONG_AI_DELAY` 環境変数（ミリ秒）で調整できます。デフォルトは `600` です。

```bash
MAHJONG_AI_DELAY=200 pnpm start   # 高速
MAHJONG_AI_DELAY=1000 pnpm start  # 低速
```

## Controls

通常の打牌:

- `←` / `→`: 手牌カーソルを移動する。端まで行くと反対側へ回り込む
- `1`–`9`: 手牌を番号で直接選択
- `Enter`: 選択中の牌を打牌
- `R`: リーチ
- `T`: ツモ
- `K`: カン（暗槓、加槓）
- `Y`: 九種九牌
- `Q`: ゲームを終了

鳴き:

- `←` / `→`: 鳴き候補を選択
- `L`: ロン
- `C`: チー
- `P`: ポン
- `K`: 大明カン
- `Space` / `Esc`: 鳴かずにパス

局終了後:

- `N` / `Enter` / `Space`: 次局へ進む

対戦終了後:

- `Space` / `Q`: 新しい対戦を開始

AI性格設定画面:

- `←` / `→`: プレイヤー選択、テンプレート変更
- `↑` / `↓`: カスタム調整時のパラメータ選択
- `Enter`: カスタム調整モードの切替
- `R`: 全プレイヤーをランダムに割り当て
- `S`: ゲーム開始
- `Q`: キャンセル

## Development

```bash
pnpm test
pnpm test:watch
pnpm exec tsc --noEmit
```

`pnpm test` は Vitest のユニットテストを一回実行します。
`pnpm exec tsc --noEmit` は TypeScript の型チェックだけを実行します。

## Project Structure

```text
src/:
  game/        牌、あがり形、役、点数、ドラ、AI（ai.ts, aiPersonality.ts）
  state/       対戦状態（types, reducer, selectors）、局進行（finishRound, claimPhase）、保存と再開
  ui/          Ink/React のターミナル UI（App, PersonalitySetup, DiscardView, WaitsInfo, KeyLegend など）
  index.tsx    エントリーポイント
docs/adr/      重要な設計判断（アーキテクチャ決定記録）
CONTEXT.md     プロジェクト用語集
```

東風戦専用のゲームとして v1.0.0 に到達しました。
半荘（東南戦）、西入、順位精算（ウマ、オカ）などへの拡張は予定していません。
