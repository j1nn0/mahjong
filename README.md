# Mahjong

React Ink と TypeScript で作る、ターミナル上の一人用日本麻雀ゲームです。

人間プレイヤー1人が、3人のコンピュータプレイヤーと東風戦を遊べることを当面の完成基準にしています。全役・全特殊裁定の完全対応よりも、まずは局進行、点棒移動、供託、本場、親継続、保存復元、最終順位まで破綻せず進むことを優先しています。

## Current Scope

実装済み、または現在の主対象:

- 1人用の四人麻雀
- 東風戦の局進行
- 親継続、オーラス親トップ終了、本場、供託
- ロン、ツモ、流局、聴牌料
- リーチ、リーチ後ツモ切り
- チー、ポン、大明槓
- ドラ、赤ドラ、裏ドラ、カンドラ
- 七対子、国士無双を含む和了形判定
- 自分の捨て牌、同巡見逃し、リーチ後見逃しによるフリテン
- コンピュータプレイヤーの自動打牌
- 起動時の保存済み対戦の復元選択
- 自分と他家の捨て牌、副露表示

当面の範囲外:

- 半荘
- ウマ、オカ
- 暗槓、加槓
- 九種九牌、四風連打、四家立直、三家和などの特殊流局
- 責任払いなどの特殊裁定
- 状態管理が必要な一発、海底、河底、嶺上開花、槍槓などの完全な実戦成立

方針の背景は [ADR 0001](docs/adr/0001-finish-east-only-game-before-full-rules.md) を参照してください。

## Requirements

- Node.js
- pnpm

## Setup

```bash
pnpm install
```

## Run

```bash
pnpm start
```

保存済み対戦がある場合、起動時に復元するか新規開始するかを選べます。保存ファイルは `.mahjong-save.json` です。

## Controls

通常の打牌:

- `←` / `→`: 手牌カーソル移動。端まで行くと反対側へ回り込みます。
- `1`-`9`: 手牌を番号で選択
- `Enter`: 選択中の牌を打牌
- `R`: リーチ
- `T`: ツモ

鳴き:

- `C`: チー
- `P`: ポン
- `K`: カン
- `Space` / `Q`: 鳴かずにパス
- `←` / `→`: 鳴き候補の選択

局終了後:

- `N` / `Enter` / `Space`: 次局へ進む

対戦終了後:

- `Space` / `Q`: 新しい対戦を開始

## Development

```bash
pnpm test
pnpm test:watch
pnpm exec tsc --noEmit
```

`pnpm test` は Vitest のユニットテストを一回実行します。`pnpm exec tsc --noEmit` は TypeScript の型チェックだけを実行します。
