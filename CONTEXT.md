# 麻雀ゲーム (mahjong)

## 用語集

### 麻雀の方式
- **日本麻雀 (Riichi Mahjong)**: 本プロジェクトで実装する麻雀のバリエーション。リーチ、ドラ、符計算、役満などを含む標準的な日本ルール。

### ゲームルール適用範囲
- **スタンダード**: 全役（約40種）・符計算・ドラ・裏ドラ・赤ドラ・カンドラ・一発・ダブルリーチ・流局・聴牌料・オーラス/トップ確定までを実装。九種九牌などの特殊な途中流局やダブロンなどは後日追加。
- **拡張可能設計**: 役や符のデータ構造は後から追加できるように設計する。

### 点数ルール
- **持ち点**: 25000点（25000点返し → 事実上30000点持ち）
- **ウマ**: 10-20（1位+20, 2位+10, 3位-10, 4位-20）
- **オカ**: あり
- **赤ドラ**: 各色1枚ずつ（萬子5・筒子5・索子5）
- **チップ**: なし
- **親の連荘条件**: 和了または聴牌流局

### 対戦方式
- **一人用 (対AI)**: 人間1名が3名のコンピュータプレイヤーと対戦する形式。

### セーブ・ロード
- **自動セーブのみ**: 各局終了時にJSONでシリアライズして保存。手動セーブUIは後日対応。

### AI (コンピュータプレイヤー)
- **ルールベースAI**: シャンテン数を計算し、最も手牌を進める牌を選択する。
- **危険牌回避**: 他家の捨て牌からスジ・壁・現物を推測し、安全な牌を優先的に切る。
- **2段階開発**: まずルールベースで動作させ、段階的に危険牌推測ロジックを追加する。

### テスト
- **vitest**: ユニットテストフレームワーク。
- **テスト対象**: `src/game/` の純粋関数（役判定、符計算、和了判定、シャンテン数計算）を網羅。統合テストとして局進行のシナリオテストも一部含める。

### 言語・実行環境
- **TypeScript**: 型安全のため。
- **tsx**: Node.js上でTypeScriptを直接実行するためのローダー。

### TUIフレームワーク
- **ink**: ReactベースのターミナルUIフレームワーク。状態管理とコンポーネント指向で牌の描画や操作を構築する。

### 操作方式
- **ハイブリッド**: カーソルキー（←→）での牌選択に加え、数字キー（1〜N）での直接選択も対応。
- **アクションキー**: c=チー, p=ポン, k=カン, r=リーチ, Enter=打牌決定, Space=メニュー, q=キャンセル/放棄。

### 画面レイアウト
- **下部集中 (パターンC)**: 画面下部に自分の手牌とアクションメニューを配置。中央に河（全員の捨て牌）、上部に対面情報をコンパクトに表示。
- **牌の表示**: Unicode麻雀牌（🀇〜🀫）を使用。

### 実装フェーズ
1. **核ロジック**: 牌・手牌・山のデータ構造、シャッフル、配牌。
2. **ゲーム進行**: ツモ・打牌・リーチ・ロン・ツモ和了の基本フロー。
3. **鳴き**: チー・ポン・カン。
4. **役判定 + 得点計算**: 全役 + 符計算 + 得点計算。
5. **ドラ系**: ドラ・裏ドラ・赤ドラ・カンドラ。
6. **AI実装**: ルールベースAI + 危険牌回避。
7. **仕上げ**: 自動セーブ、流局処理、オーラス/トップ確定、UI磨き。
